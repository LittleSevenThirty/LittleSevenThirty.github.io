介绍

npm 是 Node.js 包管理工具，可以用来安装、管理和发布 Node.js 包。本文将介绍 npm 的基本用法，帮助你快速上手 npm, 并不需要学习Node.js的知识。

npm 全称为Node Package Manager, 是一个命令行工具，用于管理Node.js 项目中的依赖包,它可以自动安装和更新项目中所需的依赖包，并在项目中生成一个package.json文件，记录项目中安装的依赖包及其版本信息.

  1. 我们准备一个文件夹（我以npmapp为例），并在该文件夹下Shirft+右键Git bash here打开命令行窗口，并右键在vscode打开此文件夹

  2. 输入npm -v或者npm –version(完整的单词前面是两个减号)查看npm版本,

  3. 输入npm help查看npm命令的帮助信息

  4. 可以输入ctrl+l清屏(其实往上滚轮还存在)，输入npm init 生成package.json文件[ 它会向我们询问package.json文件的一些信息，如项目名称、版本、描述、作者、许可证、关键字、入口文件、仓库地址等,这些可以提前设置，也可以直接回车使用默认值后期使用命令修改 ]，并自动生成默认的package.json文件内容(package.json文件记录了项目中安装的依赖包及其版本信息)

使用npm init -y或者npm init –yes命令可以跳过向我们询问package.json文件的过程，直接生成默认的package.json文件内容。

配置键 描述 示例
registry 指定 npm 注册表的 URL。 npm config set registry https://registry.npmjs.org/
cache 指定 npm 缓存目录的位置。 npm config set cache /path/to/cache
prefix 指定全局安装包的前缀路径。 npm config set prefix /usr/local
loglevel 设置日志级别，如 silent, error, warn, info, verbose, silly。 npm config set loglevel info
init-author-name 设置初始化新项目时的默认作者姓名。 npm config set init-author-name "Your Name"
init-author-email 设置初始化新项目时的默认作者邮箱。 npm config set init-author-email "you@example.com"
init-author-url 设置初始化新项目时的默认作者网址。 npm config set init-author-url "http://yourwebsite.com"
init-version 设置初始化新项目时的默认版本号。 npm config set init-version "1.0.0"
init-license 设置初始化新项目时的默认许可证。 npm config set init-license "MIT"
save-exact 控制是否在 package.json 中保存精确的版本号。 npm config set save-exact true
save-prefix 设置保存依赖版本时的前缀，如 ^ 或 ~。 npm config set save-prefix "^"
strict-peer-dependencies 控制是否严格检查 peer dependencies。 npm config set strict-peer-dependencies false
ignore-scripts 控制是否忽略脚本。 npm config set ignore-scripts true
scripts-prepend-node-path 控制是否在运行脚本时将 Node.js 可执行文件的路径添加到 PATH 环境变量中。 npm config set scripts-prepend-node-path warn
proxy 设置 HTTP 代理服务器。 npm config set proxy http://proxy.example.com:8080
https-proxy 设置 HTTPS 代理服务器。 npm config set https-proxy http://proxy.example.com:8080
cafile 指定 CA 文件的路径，用于 HTTPS 请求。 npm config set cafile /path/to/cafile.pem

npm 下载局部模块

如果不指明是 -g 或 –global 则都是下载局部模块，即只适用于当前文件夹
-s 则指定下载模块后是否更新package.json里的内容

下载依赖包

  • npm install:
    默认情况下，这个命令会安装 package.json 文件中列出的所有依赖项，包括开发环境依赖（devDependencies）和生产环境依赖（dependencies）。
    它适用于开发阶段，因为开发人员通常需要所有类型的依赖来运行测试、构建项目等。
  • npm install –production 或者在 .npmrc 文件中设置了 production=true：
    这个命令只会安装 dependencies 中列出的依赖项，而不会安装 devDependencies 中的依赖。
    它主要用于生产环境部署，当你不需要开发工具或者测试框架等只在开发过程中使用的工具时，使用这个选项可以减少不必要的包安装，从而节省时间和磁盘空间。
